01

Semantic Modelling

Power BI · Tabular · Star Schema

RGBI designs Power BI semantic models as governed intelligence systems — not collections of tables wired together reactively. Every model begins with grain definition, relationship integrity checks, and a clear separation between fact, dimension, and bridge structures.

The result is a model that scales — adding measures, changing report layouts, or onboarding new consumers — without requiring re-architecture. What most practitioners discover at report-build time, RGBI resolves at design time.

Delivered capabilities

Star and snowflake schema architecture · Relationship cardinality and cross-filter direction control · Bridge table design for many-to-many resolution · Calculated column vs. measure decision framework · Import vs. DirectQuery evaluation · Date/calendar table governance · Role-level security implementation

02

DAX Engineering

Measures · Evaluation Context · Performance

DAX is not a formula language — it is an evaluation engine with precise context rules that most practitioners learn by trial and error. RGBI engineers DAX measures from first principles: filter context is stated explicitly, ALLSELECTED and ALL are never used interchangeably, and every measure is tested against known edge cases before delivery.

Measures are structured within a governed hierarchy, carrying descriptions, display folder assignments, and format strings. A measure without documentation is a liability in any governed environment.

Delivered capabilities

Time intelligence (fiscal and calendar) · Semi-additive measures · Running totals and moving averages · Statistical measures (Z-scores, standard deviation, control limits) · Context transition management · CALCULATE and filter modifier patterns · Measure dependency mapping · Performance profiling and query optimisation

Technologies

03

Statistical Process Control Frameworks

SPC · Control Charts · Anomaly Detection

Operational performance monitoring built on control chart theory produces thresholds that reflect the process, not the opinion of a stakeholder in a workshop. RGBI implements SPC frameworks where control limits are derived from the data's own statistical distribution — Z-score logic with explicit one-sided bad-direction classification, volatility-adjusted bounds, and Poisson modelling for rare-event detection.

The output is a monitoring system where an alert means something — distinguishing genuine process shift from natural variation. Arbitrary RAG thresholds are the low-bar alternative RGBI replaces.

Delivered capabilities

X-bar and individuals control charts · UCL/LCL derivation from sigma multiples · Trend and run detection (Nelson/Western Electric rules) · Poisson rate modelling for discrete event streams · Z-score normalisation across heterogeneous KPIs · Bayesian updating for small sample correction · SPC DAX measure patterns · Executive SPC narrative layers

Technologies

Power BI DAX Python (pandas, scipy) Azure SQL
04

DAMA DMBOK Governance

Platinum Tier · Metadata · Lineage · Stewardship

RGBI implements data governance aligned to DAMA DMBOK 2.0 Platinum standards — not as a documentation exercise, but as an embedded design requirement in the semantic model itself. Every measure carries a data owner, data steward, and computational lineage annotation. Every table has a defined grain and agreed business definition.

The Platinum Assurance Framework ensures that every governed measure can be traced from source data through transformation logic to its presentation in a report visual — without relying on institutional memory to fill the gaps.

Delivered capabilities

Measure metadata catalogue (owner, steward, description, lineage) · Display folder taxonomies aligned to governance domains · Data dictionary and business glossary integration · Grain-level table documentation · Quality assurance annotation patterns · Change management protocols for governed models · Client handover and knowledge transfer programmes

Technologies

Tabular Editor 3 Power BI SharePoint Microsoft Purview

Agentic AI Delivery

MCP Toolchains · Claude API · Governed Automation

AI augmentation in model development is only valuable when it accelerates correctness, not when it generates plausible-looking output that later fails under scrutiny. RGBI integrates agentic workflows — using MCP toolchains, the Claude API, and Power BI modelling MCPs — to automate repeatable governance tasks, measure scaffolding, and model documentation.

Every AI-assisted output is validated against the same governance standard as manually authored work. The acceleration is in delivery throughput, not in the quality threshold.

Delivered capabilities

Automated measure scaffolding with governance metadata pre-populated · MCP-integrated semantic model interrogation and audit · AI-assisted data dictionary generation and maintenance · Agentic quality assurance workflows for model validation · Claude API integration for narrative intelligence layers · VS Code and Tabular Editor toolchain automation
