Analytics Platform

Transaction Intelligence

A high-volume transactional analytics platform engineered for organisations processing large volumes of discrete events — where near-real-time performance monitoring, anomaly detection, and statistical process control are operational requirements, not reporting enhancements.

The platform monitors 1M+ monthly events across multiple performance domains, applying SPC-derived control limits to distinguish genuine process deviation from natural variation. Alerts are statistically grounded — not arbitrary thresholds agreed in a workshop.

Applicable sectors

Manufacturing (sensor and event-level production data) · Payment processors and fintech platforms · Public transport ticketing systems · Retail and point-of-sale transaction networks · Telecommunications (call and data event records) · Toll and road infrastructure operators · Utilities (metering and consumption events)

1M+ Monthly discrete events monitored
Near real-time SPC monitoring cadence
Multi-domain Performance tracking scope
Platinum RGBI governance standard

The Problem

Organisations processing high-frequency transactional event streams typically monitor performance through lagged aggregate reports — daily or weekly rollups that obscure intra-period anomalies until they become operational incidents. Threshold-based alerting, where it exists, relies on manually-set limits that do not reflect the process's actual statistical behaviour.

The result: genuine performance deviation goes undetected until it accumulates into a pattern that is visible in a standard report — by which point recovery cost has compounded.

The Solution

RGBI designed a semantic model with SPC-derived control limits calculated directly from the event stream's statistical distribution. X-bar and individuals control chart logic was implemented in DAX, with UCL and LCL recalculating dynamically as the data window advances.

Anomaly detection distinguishes between common-cause variation and assignable causes, flagging only genuine process shifts. The governed measure hierarchy ensures every alert is traceable from source event through calculation logic to the flag presented in the monitoring dashboard.

Technical Architecture

  • Power BI semantic model with star-schema event fact table
  • DAX SPC measures: mean, sigma, UCL, LCL, violation flags
  • Near-real-time refresh via Azure SQL event staging layer
  • Python ETL pipeline using pyodbc and fast_executemany
  • Multi-domain performance dimensions for cross-domain analysis
  • Role-level security aligned to operational scope boundaries
  • DAMA DMBOK Platinum metadata on all governed measures

Governance Outcomes

  • 1,000+ governed measures across operational domains
  • Every alert measure carries data owner and steward annotation
  • Computational lineage documented from source to visual
  • SPC methodology documented and peer-reviewed
  • Threshold derivation auditable — no arbitrary limits
  • Model handover with client knowledge transfer programme
Commercial Intelligence

Commercial Performance Intelligence

A governed commercial analytics platform for organisations managing order-to-delivery performance, fulfilment accuracy, and KPI governance across multiple business domains. Built for environments where commercial performance data is fragmented across systems and stakeholders demand a single, auditable source of truth.

The platform covers 190+ governed measures across 13 commercial domains, providing end-to-end visibility from order receipt through invoice reconciliation — with statistical defensibility at every layer.

Applicable sectors

B2B distribution and wholesale · FMCG manufacturers · Industrial and equipment distributors · Logistics and third-party logistics providers · Retail supply chains

190+ Governed measures delivered
13 Commercial domains covered
OTIF Fulfilment framework basis
Platinum RGBI governance standard

The Problem

Commercial organisations typically measure fulfilment performance through surface-level KPIs — on-time delivery rates and fill rates calculated from a single data source — without reconciling against order, invoice, and dispatch records. The result is a metric that feels credible but cannot withstand challenge when a customer disputes performance.

Governance is often absent: no agreed business definitions, no documented calculation logic, and no clear ownership. When the analyst who built the report leaves, institutional knowledge leaves with them.

The Solution

RGBI designed a governed semantic model covering the full order-to-invoice lifecycle. Order fulfilment timeliness and accuracy — the OTIF framework — was implemented with statistically derived performance bands rather than arbitrary targets. Reconciliation logic ensures that order, dispatch, and invoice data are consistently aligned before any KPI is calculated.

A layered computational architecture separates base transaction metrics from domain-level aggregations and executive summary narratives — ensuring that every board-level figure is traceable to a source transaction.

Technical Architecture

  • Power BI semantic model: order, dispatch, and invoice fact tables
  • 4-4-5 fiscal calendar with period-comparison DAX measures
  • OTIF measure suite: timeliness, accuracy, and composite scores
  • Cross-domain bridge table for multi-entity analysis
  • Azure SQL staging layer with Python ETL and Entra ID auth
  • 13 commercial display folder domains in governed measure library
  • Executive narrative layer with period-over-period commentary logic

Governance Outcomes

  • 190+ measures with full DAMA DMBOK Platinum metadata
  • Business definitions agreed and documented per domain
  • Data owner and steward assigned to every governed measure
  • Reconciliation audit trail from order to invoice
  • OTIF methodology peer-reviewed and signed off by client
  • Knowledge transfer programme enabling client team independence
